What IFP Does

IFP v2.0 is Anaplan's application suite for the Office of the CFO — covering P&L, cash flow, and balance sheet planning through a set of modules built on the Anaplan Application Framework and Polaris.

Problems IFP Solves

ProblemHow IFP Solves It
Inefficient Planning CyclesPre-built driver-based forecasting methods automate routine calculations
Fragmented Enterprise PlanningSingle connected platform — Revenue, OpEx, HC, CapEx, and BS in one place
Reactive DecisionsBuilt-in scenario modeling and what-if analysis
Long implementation lead timeDeploy in 8–10 weeks using Application Framework best practices
Rigid SystemsConfigurable dimensions, planning methods, and extensible architecture

Six Planning Modules

ModuleWhat It Plans
💰 Revenue & COGSRevenue and margins by product, customer, entity — driver-based methods
📊 Operating ExpensesOpEx by GL account — planning methods, line item detail, allocations
👥 HeadcountJob-level workforce costs — pay bands, HC changes, cost sync to FP
🏗 CapExIndividual assets — depreciation, disposals, P&L and BS impact
📋 Balance Sheet & CFWorking capital, funding logic, indirect cash flow — activity-based
🎯 Top-Down PlanningExecutive targets for revenue, margin, OpEx, FTE — vs. bottom-up

Plus: Reporting & Analysis — consolidated IS, BS, CF reports, variance analysis, statistical analysis, and management reporting pack.

What's New in v2.0

FeatureWhat It Means
Anaplan Application FrameworkUpgradeable, centrally-deployed app — replaces static v1.x configurator
Anaplan Data Orchestrator (ADO)Replaces the data hub model — lightweight pipelines with no memory overhead in the workspace
Job-Based HeadcountPlans at role level, not employee — protects PII, enables OWP integration
Progressive Disclosure UIShows only relevant inputs per planning method — reduces errors
Re-architected ModelsPolaris engine: optimized for large data volumes compared to the Classic engine
🚨 Customer Exclusion Criteria

IFP v2.0 is NOT suitable for: customers requiring BYOK on public cloud, customers who cannot use AWS (ADO requirement), public sector/federal organizations, or customers requiring more than 8 dimensions.
